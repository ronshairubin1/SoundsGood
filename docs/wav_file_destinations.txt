# .wav File Locations in Codebase

This file shows where .wav files are read from and written to in the codebase.

## Sound File Directories

| Directory | Purpose | Operations | References |
|-----------|---------|------------|------------|
| data/sounds/raw_sounds/ | Storage for original unprocessed recordings before chunking | WRITE --> | 1 |
| data/sounds/raw_sounds/{class}/ | Class-specific directories for raw recordings | WRITE --> | 0 |
| data/sounds/temp_sounds/ | Temporary storage for sound chunks during processing | WRITE --> | 0 |
| data/sounds/temp_sounds/{class}/ | Class-specific directories for temporary sound chunks | WRITE --> | 0 |
| data/sounds/training_sounds/ | Main sound storage directory containing verified sound files used for training models | READ, WRITE <--> | 14 |
| data/sounds/training_sounds/{class}/ | Class-specific directories containing sound samples for each class (e.g., 'ah', 'eh', etc.) | READ, WRITE <--> | 0 |
| root/ | Root directory - used mainly for test recordings | WRITE --> | 4 |
| root/temp/ | Temporary files for ML operations (predictions, processing) | WRITE, READ <--> | 13 |

## File Operations Explained

### Reading Operations (Files Sourced From)

- **data/sounds/training_sounds/** <--> .wav
  - Purpose: Main sound storage directory containing verified sound files used for training models
  - Referenced as: Config.TRAINING_SOUNDS_DIR, Config.SOUNDS_DIR, class_path

- **data/sounds/training_sounds/{class}/** <--> .wav
  - Purpose: Class-specific directories containing sound samples for each class (e.g., 'ah', 'eh', etc.)
  - Referenced as: os.path.join(Config.TRAINING_SOUNDS_DIR, class_name)

- **root/temp/** <--> .wav
  - Purpose: Temporary files for ML operations (predictions, processing)
  - Referenced as: Config.TEMP_DIR, temp_path

### Writing Operations (Files Written To)

- **data/sounds/raw_sounds/** --> .wav
  - Purpose: Storage for original unprocessed recordings before chunking
  - Referenced as: Config.RAW_SOUNDS_DIR, raw_sounds_dir

- **data/sounds/raw_sounds/{class}/** --> .wav
  - Purpose: Class-specific directories for raw recordings
  - Referenced as: os.path.join(Config.RAW_SOUNDS_DIR, class_name)

- **data/sounds/temp_sounds/** --> .wav
  - Purpose: Temporary storage for sound chunks during processing
  - Referenced as: Config.TEMP_SOUNDS_DIR, temp_sounds_dir

- **data/sounds/temp_sounds/{class}/** --> .wav
  - Purpose: Class-specific directories for temporary sound chunks
  - Referenced as: os.path.join(Config.TEMP_SOUNDS_DIR, class_name)

- **data/sounds/training_sounds/** <--> .wav
  - Purpose: Main sound storage directory containing verified sound files used for training models
  - Referenced as: Config.TRAINING_SOUNDS_DIR, Config.SOUNDS_DIR, class_path

- **data/sounds/training_sounds/{class}/** <--> .wav
  - Purpose: Class-specific directories containing sound samples for each class (e.g., 'ah', 'eh', etc.)
  - Referenced as: os.path.join(Config.TRAINING_SOUNDS_DIR, class_name)

- **root/** --> .wav
  - Purpose: Root directory - used mainly for test recordings
  - Referenced as: test_recording.wav

- **root/temp/** <--> .wav
  - Purpose: Temporary files for ML operations (predictions, processing)
